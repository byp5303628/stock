| 维度 | 内容 |
|------|------|
| 接口总数 | 12 个 |
| 接口用例数 | 38 个（正常 22 / 异常 16） |

| 覆盖模块 | stock-common, stock-core, stock-biz |

## 接口测试

### API-001 模型列表

- **路径**: `GET /api/metadata/model/list.json`
- **方法**: GET
- **描述**: 查询所有元数据模型列表

#### 用例 1: 正常查询（系统中有模型）

- **说明**: 数据库中存在多个模型时，返回完整列表

- **curl 命令**:
  ```bash
  curl -X GET 'http://localhost:8080/api/metadata/model/list.json'
  ```

- **预期状态码**: `200`

- **预期响应**:
  ```json
  {
    "success": true,
    "data": [
      {
        "id": 1,
        "name": "市盈率指标",
        "code": "pe_metric",
        "modelType": "INDICATOR",
        "description": "市盈率相关指标字段定义",
        "status": "PUBLISHED"
      },
      {
        "id": 2,
        "name": "交易参数配置",
        "code": "trade_config",
        "modelType": "CONFIG",
        "description": "交易参数配置字段定义",
        "status": "DRAFT"
      }
    ],
    "errorMsg": null
  }
  ```

- **验证点**:
  - [ ] 状态码为 200
  - [ ] success 为 true
  - [ ] data 为数组，每个元素包含 id/name/code/modelType/status
  - [ ] 返回顺序按 gmt_modified 倒序

#### 用例 2: 正常查询（系统中无模型）

- **说明**: 数据库为空时，返回空列表

- **curl 命令**:
  ```bash
  curl -X GET 'http://localhost:8080/api/metadata/model/list.json'
  ```

- **预期状态码**: `200`

- **预期响应**:
  ```json
  {
    "success": true,
    "data": [],
    "errorMsg": null
  }
  ```

- **验证点**:
  - [ ] data 为空数组而非 null

### API-002 模型详情

- **路径**: `GET /api/metadata/model/detail.json`
- **方法**: GET
- **描述**: 查询单个元数据模型的完整信息（含字段列表）

#### 用例 1: 正常查询（模型存在）

- **说明**: 按 ID 查询存在的模型，返回模型信息及所有字段

- **请求参数**: `?id=1`

- **curl 命令**:
  ```bash
  curl -X GET 'http://localhost:8080/api/metadata/model/detail.json?id=1'
  ```

- **预期状态码**: `200`

- **预期响应**:
  ```json
  {
    "success": true,
    "data": {
      "id": 1,
      "name": "市盈率指标",
      "code": "pe_metric",
      "modelType": "INDICATOR",
      "description": "市盈率相关指标字段定义",
      "status": "PUBLISHED",
      "fields": [
        {
          "id": 1,
          "fieldName": "period",
          "fieldType": "ENUM",
          "businessMeaning": "财报周期，如 Q1/Q2/Q3/Q4",
          "required": true,
          "enumId": 1,
          "sortOrder": 1
        },
        {
          "id": 2,
          "fieldName": "pe_value",
          "fieldType": "NUMBER",
          "businessMeaning": "市盈率数值",
          "required": true,
          "constraints": "{\"min\": 0}",
          "sortOrder": 2
        }
      ]
    },
    "errorMsg": null
  }
  ```

- **验证点**:
  - [ ] 返回的 id 与请求一致
  - [ ] fields 数组包含该模型的所有字段
  - [ ] 字段按 sortOrder 升序排列

#### 用例 2: 模型不存在

- **说明**: 查询不存在的模型 ID

- **请求参数**: `?id=99999`

- **curl 命令**:

- **预期状态码**: `200`

- **预期响应**:
  ```json

    "success": false,
    "data": null,
    "errorMsg": "模型不存在"
  }
  ```

- **验证点**:
  - [ ] success 为 false
  - [ ] errorMsg 包含错误描述

#### 用例 3: 参数缺失

- **说明**: 未传入 id 参数

### API-003 新建/更新模型

- **路径**: `POST /api/metadata/model/save.json`
- **方法**: POST
- **描述**: 新建元数据模型；如果传入 id 则为更新

#### 用例 1: 新建模型（正常流程）

- **说明**: 创建一个新的元数据模型

- **请求参数**:
  ```json
  {
    "name": "营收指标",
    "code": "revenue_metric",
    "modelType": "INDICATOR",
    "description": "营收相关指标字段定义"
  }
  ```

- **curl 命令**:
  ```bash
  curl -X POST 'http://localhost:8080/api/metadata/model/save.json' \
    -H 'Content-Type: application/json' \
    -d '{"name":"营收指标","code":"revenue_metric","modelType":"INDICATOR","description":"营收相关指标字段定义"}'
  ```

- **预期状态码**: `200`

- **预期响应**:
  ```json
  {
    "success": true,
    "data": {
      "id": 3,
      "name": "营收指标",
      "code": "revenue_metric",
      "modelType": "INDICATOR",
      "status": "DRAFT"
    },
    "errorMsg": null
  }
  ```

- **验证点**:
  - [ ] 返回的 id 不为空
  - [ ] 新建的模型状态默认为 DRAFT
  - [ ] 数据库中插入一条记录，code 唯一

#### 用例 2: 更新模型（正常流程）

- **说明**: 更新已有模型的描述信息

- **请求参数**:
  ```json
  {
    "id": 3,
    "name": "营收指标",
    "code": "revenue_metric",
    "modelType": "INDICATOR",
    "description": "更新后的营收指标描述"
  }
  ```

- **curl 命令**:
  ```bash
  curl -X POST 'http://localhost:8080/api/metadata/model/save.json' \
    -H 'Content-Type: application/json' \
    -d '{"id":3,"name":"营收指标","code":"revenue_metric","modelType":"INDICATOR","description":"更新后的营收指标描述"}'
  ```

- **预期状态码**: `200`

- **验证点**:
  - [ ] description 被更新
  - [ ] gmt_modified 更新为新时间
  - [ ] id、name、code 保持不变

#### 用例 3: 重复 code

- **说明**: 创建模型时使用已存在的 code

- **请求参数**:
  ```json
  {
    "name": "另一个市盈率",
    "code": "pe_metric",
    "modelType": "INDICATOR"
  }
  ```

- **curl 命令**:
  ```bash
  curl -X POST 'http://localhost:8080/api/metadata/model/save.json' \
    -H 'Content-Type: application/json' \
    -d '{"name":"另一个市盈率","code":"pe_metric","modelType":"INDICATOR"}'
  ```

- **预期状态码**: `200`

- **预期响应**:
  ```json
  {
    "success": false,
    "data": null,
    "errorMsg": "模型编码 pe_metric 已存在"
  }
  ```

- **验证点**:
  - [ ] 重复 code 时返回明确的错误信息

#### 用例 4: 缺少必填字段

- **说明**: 未传入 name

- **请求参数**:
  ```json
  {
    "code": "test_model",
    "modelType": "INDICATOR"
  }
  ```

- **curl 命令**:
  ```bash
  curl -X POST 'http://localhost:8080/api/metadata/model/save.json' \
    -H 'Content-Type: application/json' \
    -d '{"code":"test_model","modelType":"INDICATOR"}'
  ```

- **预期状态码**: `400`

- **验证点**: 返回参数校验错误

### API-004 Schema 校验

- **路径**: `POST /api/metadata/model/validate.json`
- **方法**: POST
- **描述**: 对指定模型执行 Schema 校验

#### 用例 1: 校验通过（所有字段合法）

- **说明**: 模型的字段定义全部合法，校验通过

- **请求参数**:
  ```json
  {
    "modelId": 1
  }
  ```

- **curl 命令**:
  ```bash
  curl -X POST 'http://localhost:8080/api/metadata/model/validate.json' \
    -H 'Content-Type: application/json' \
    -d '{"modelId":1}'
  ```

- **预期状态码**: `200`

- **预期响应**:
  ```json
  {
    "success": true,
    "data": {
      "valid": true,
      "errors": []
    },
    "errorMsg": null
  }
  ```

- **验证点**:
  - [ ] valid 为 true
  - [ ] 模型状态从 DRAFT 变更为 PUBLISHED

#### 用例 2: 校验失败（字段缺少 business_meaning）

- **说明**: 模型中存在 required 字段但没有填写 business_meaning

- **请求参数**:
  ```json
  {
    "modelId": 2
  }
  ```

- **预期响应**:
  ```json
  {
    "success": true,
    "data": {
      "valid": false,
      "errors": [
        {"field": "period", "message": "必填字段缺少业务含义描述"}
      ]
    },
    "errorMsg": null
  }
  ```

- **验证点**:
  - [ ] valid 为 false
  - [ ] errors 中包含每个不合规字段的详细信息
  - [ ] 模型状态保持 DRAFT 不变

#### 用例 3: 校验失败（引用的枚举已被删除）

- **说明**: 字段绑定的枚举已被软删除

- **请求参数**:
  ```json
  {
    "modelId": 3
  }
  ```

- **预期响应**:
  ```json
  {
    "success": true,
    "data": {
      "valid": false,
      "errors": [
        {"field": "period", "message": "字段引用的枚举（id=1）已被删除"}
      ]
    },
    "errorMsg": null
  }
  ```

- **验证点**: 引用的枚举状态为 DELETED 时校验失败

#### 用例 4: 模型不存在

- **请求参数**:
  ```json
  {
    "modelId": 99999
  }
  ```

- **预期响应**:
  ```json
  {
    "success": false,
    "data": null,
    "errorMsg": "模型不存在"
  }
  ```

### API-008 新建/更新枚举

- **路径**: `POST /api/metadata/enum/save.json`
- **方法**: POST
- **描述**: 新增或更新枚举定义（含枚举值）

#### 用例 1: 新建枚举（正常流程）

- **说明**: 新建一个枚举并定义枚举值

- **请求参数**:
  ```json
  {
    "name": "业务线",
    "code": "biz_line",
    "description": "业务线枚举",
    "values": [
      {"valueCode": "RETAIL", "valueLabel": "零售业务", "sortOrder": 1},
      {"valueCode": "CORP", "valueLabel": "对公业务", "sortOrder": 2}
    ]
  }
  ```

- **curl 命令**:
  ```bash
  curl -X POST 'http://localhost:8080/api/metadata/enum/save.json' \
    -H 'Content-Type: application/json' \
    -d '{"name":"业务线","code":"biz_line","description":"业务线枚举","values":[{"valueCode":"RETAIL","valueLabel":"零售业务","sortOrder":1},{"valueCode":"CORP","valueLabel":"对公业务","sortOrder":2}]}'
  ```

- **验证点**:
  - [ ] 枚举创建成功
  - [ ] 枚举值一并写入 metadata_enum_value 表

  ```json
  {
    "name": "重复周期",
    "code": "report_period"
  }
  ```

- **预期响应**:
  ```json
  {
    "success": false,
    "data": null,
    "errorMsg": "枚举编码 report_period 已存在"
  }
  ```

#### 用例 3: 更新枚举时枚举值清空

- **说明**: 更新枚举时如果 values 未传入，不修改已有枚举值

- **请求参数**:
